 Γράφω μια διπλωματική εργασία επιπέδου Master σε markdown, στο αρχείο gkinis_konstantinos.md (από δω και πέρα θα το λέω "main")
    σχετικά με τη δουλειά μου στην εταιρεία Maersk, στην ομάδα NetZero που      
    ασχολείται με τις εκπομπές αερίων θερμοκηπίου από τον κύκλο εργασιών της    
    εταιρείας. 
    
Μπορείς να δεις παρουσιάσεις που δώσαμε σχετικά με τη δουλειά μας, στο directory `./presentations/` και να περιγράψεις το σκοπό και τη μεθοδολογία μας.
    Ο κώδικας του λογισμικού που αναλύει η διπλωματική είναι στο    `./nrg` directory.
  To `./nrg/guides/` directory  έχει το μεγαλύτερο μέρος του documentation.
  Ο κώδικας είναι χωρισμένος σε τμήματα στο `./nrg/components/` directory. Εκεί μπορείς να βρεις Phoenix framework live views, που διαβάζοντας την HTML τους μπορείς να καταλάβεις το σκοπό τους. Επίσης μπορείς να διαβάσεις τις περιγραφές των tests για το κάθε live view.

  Έχω ήδη γράψει ένα τμήμα της διπλωματικής, στο αρχείο main, μπορείς να εμπλουτίσεις αυτό.
  Το τελικό κείμενο πρέπει να στοχεύει τις 80 με 100 σελίδες Α4 σε 12pt font size, περιλαμβανομένων εικόνων και τμημάτων κώδικα.

Ο πρώτος στόχος είναι να δημιουργήσεις ένα πλάνο. Αυτό θα προκύψει από την ανάλυση των παρουσιάσεων και του κώδικα. Μπορείς επίσης να περιλάβεις πληροφορίες από το διαδύκτιο σχετικά με τις εμπλεκόμενες τεχνολογίες, μεθόδους ανάπτυξης λογισμικού, μεθόδους μέτρησης εκπομπών αερίων θερμοκηπίου κτλ. Προτείνω να στοχεύσεις στον επιμερισμό στα εξής κεφάλαια:

# Εισαγωγή
# Η κατάσταση πριν τη δημιουργία του λογισμικού
# Οι τρόποι μέτρησης εκπομπών
# Βελτιστοποίηση εκπομπών
# Βελτιστοποίηση ανεφοδιασμού καυσίμων (bunker optimization)
# Ακριβής καταγραφή εκπομπών της μηχανής του πλοίου μέσω του STAR Connect 
# Οι τεχνολογίες που χρησιμοποιήθηκαν
# Αρχιτεκτονική του συστήματος
# Εργασιακές μέθοδοι (Agile, Extreme Programming, TDD, Pair Programming, Feedback loops)
# Συμπεράσματα
# Μελλοντική Εργασία
# Βιβλιογραφία

Μπορείς να προσθέσεις, αφαιρέσεις και ομαδοποιήσεις κεφάλαια, αυτά αποτελούν απλά μια αφετηρία.

Επειδή η δημιουργία του πλάνου μπορεί να είναι πολύ απαιτητική στο token quota, θα ήθελα να την καταγράφεις σε ένα αρχείο καθώς κάνεις πρόοδο, και να το κάνεις commit στο git ώστε να μπορεί να συνεχιστεί αν χρειαστεί να σταματήσεις στη μέση λόγω κατανάλωσης tokens.

Το δεύτερο βήμα είναι η συγγραφή των κεφαλαίων. Γι'αυτό το σκοπό μπορείς να δημιουργήσεις ένα sub agent για κάθε κεφάλαιο, με το μοντέλο Sonnet, με στόχο την υλοποίηση του πλάνου. Θα πρέπει να διατηρεί το ύφος κειμένου διπλωματικής εργασίας επιπέδου master πρωτίστως και κατά το δυνατό να μοιάζει με το ήδη υπάρχον κείμενο στο `gkinis_konstantinos.md`. Ο κάθε συγγραφέας θα δουλεύει στο δικό του τμήμα του αρχικού πλάνου. Ιδανικά θα πρέπει να κάνει commit τις προσθήκες του σε τακτά χρονικά διαστήματα, κάθε φορά που ολοκληρώνει ένα υποκεφάλαιο.

Όλα αυτά πρέπει να προστεθούν στο αρχείο `gkinis_konstantinos.md` είτε ως νέα κεφάλαια, είτε επεκτείνοντας τα υπάρχοντα.
